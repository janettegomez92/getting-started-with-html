5 WAYS PROGRAMMERS THINK

1. Procedural Thinking - By finding a solution step by step 1-2-3.
2. Abstract Thinking - By using concepts 1+1=2.
3. Systematic Debugging - Finding the main cause to a problem and fixing it.
4. Technological Empathy - How a computer thinks.
5. Systems Thinking - Taking a big idea and breaking it down.

DEFINITIONS

Webpage:
A web page is a text document written in a language called HTML (Hypertext Markup Language).

Coding:
Coding is what happens when programmers write text in a language that a computer can understand.

Tag:
A tag is a command. ALL tags have opening tags, but some do not have closing tags, these are called "void tags". See some examples of tags below.

Inline Tags vs. Block Tags:
Inline tags enclose content tightly. Block tags enclose content with a block creating a paragraph look.

Element:
An Element is everything in between a set of opening and closing tags.

Attribute:
An attribute is a property of an HTML element which provides additional information about the data within the element. For example: HREF (Hypertext REFerence)

TAGS

Bold (b) Tag:
This tag makes the content in between the angled brackets bold.

Emphasis (em) Tag:
This tag makes the content in between the angled brackets italicized.

Anchor (a) Tag:
This tag defines a hyperlink. This tag will require an attribute which is href.
Dont know what a hyperlink is? Click here to find out!

Image (img) Tag:
This tag defines an image. This tag should always be followed by alternative (alt) text which will show if the image does not load.

Brake (br) Tag:
This tag basically says "line ends here". This is one of the "void tags"mentioned above.

Paragraph (p) Tag:
This tag creates a new paragraph.
